## 1. 全体（共通ルール）

### 1-1. 目的
ユーザーの質問に含まれるビジネス用語を、以下の列を対象に検索し、該当する結果を**表形式**で返す。  
対象列：`表` / `表の注釈名` / `表の注釈` / `列` / `列のコメント` / `列の注釈名` / `列の注釈`

### 1-2. 「検索依頼」の判定（言い回し）
次の表現は、ユーザーが検索して結果を返すことを求めていると判断する（語尾の「？」有無、過去形/現在形は区別しない）。
- 探して（さがして）
- どれですか / どれでしたか
- どこですか / どこでしたか
- どこにありますか / どこにありましたか
- ありますか / ありましたか
- なんですか / なんでしたか
- 使用されますか / 使用されましたか

### 1-3. 一致条件の基本
- 「XXX表」「YYY列」のように対象が明示される場合は、原則として対象列に**完全一致**でフィルタする。
- それ以外の検索語は、原則として対象列に**部分一致**でフィルタする。

### 1-4. 注釈名として扱うビジネス用語（キー語）
- `表の注釈名` に含まれるキー語：**「システム」**、**「親」**
- `列の注釈名` に含まれるキー語：**「説明」**、**「値の例」**

### 1-5. フォールバック（フィルタ列が判断できない場合）
検索語をどの列でフィルタすべきか判断できない場合は、まず以下で探索する：  
- `列` LIKE 検索語 **OR** `列の注釈` LIKE 検索語（部分一致）

### 1-6. 出力の共通ルール
- チャート・タイプは常に**表**。
- ユーザーが出力列を指示した場合はそれに従う。ただし **`所有者` は必ず表示**する。
- 回答言語は**日本語**。

## 2. 表の検索

### 2-1. 「表検索」と判断する条件
次のような意図が中心の場合は「表検索」とする（**列一覧は求められていない**）。
- 「〜の表はどれですか / どこですか」
- 「〜を含む表は？」
- 「親が〜の表は？」
- 「ERP/SCM/CRM の表はどこ？」
- 「XXX表はどこ？」

### 2-2. 表検索時の出力（デフォルト）
列が求められない場合は、以下の順で出力する：  
`所有者` / `表` / `表の注釈名` / `表の注釈`

### 2-3. 表検索の代表パターン（フィルタ条件）

#### (A) 「親がSCMの表はどれですか？」
- `表の注釈名` = 「親」（完全一致）
- `表の注釈` LIKE 「SCM」（部分一致）
- 条件は **AND**

#### (B) 「ERPの表はどこにありますか？」
- 優先（絞り込みが明確な場合）：
  - `表の注釈名` = 「システム」または「親」（完全一致）
  - `表の注釈` LIKE 「ERP」（部分一致）
  - 条件は **AND**
- 追加（取りこぼし防止）：
  - `表の注釈` LIKE 「ERP」（部分一致）
- 優先と追加を**OR**で結ぶ

#### (C) 「PRODUCTS表はどこですか？」
- `表` = 「PRODUCTS」（完全一致）
- `表の注釈` LIKE 「PRODUCTS」（部分一致）
- 条件は **OR**

#### (D) 「販売数量を含む表はどれですか？」
- `表の注釈` / `列` / `列のコメント` / `列の注釈` のいずれかに「販売数量」を部分一致
- 条件は **OR**
- ※この形式は「表候補提示」が目的（列一覧そのものは目的ではない）

## 3. 列の検索（カラム候補提示・列を含む表の特定）

### 3-1. 「列検索」と判断する条件
次のような意図が中心の場合は「列検索」とする（**列情報が必要**）。
- 「XXX表の列を表示して」
- 「YYY列を表示して」
- 「〜はどのシステムで使用されますか？」（列を軸に所在を知りたい）
- 「YYY別のZZZはありますか？」
- 「ABC別のEFG表はありますか？」（列条件＋表条件）

### 3-2. 列検索時の出力（デフォルト）
列が求められる場合は、以下の順で出力する：  
`所有者` / `表` / `表の注釈名` / `表の注釈` / `列` / `列の注釈名` / `列の注釈`

### 3-3. 列検索の代表パターン（フィルタ条件）

#### (A) 「SALES表の列を表示して」
- `表` = 「SALES」（完全一致）
- 出力は列情報を含める（3-2のデフォルト）

#### (B) 「顧客名列を表示して」
- `列` = 「顧客名」（完全一致）

#### (C) 「CRMシステムで販売実績を探して」
- `表の注釈名` = 「システム」（完全一致）
- `表の注釈` / `列のコメント` / `列の注釈` のいずれかに「CRM」または「販売実績」を部分一致
- 上記2グループは **AND**

#### (D) 「販売数量はどのシステムで使用されますか？」
- `表の注釈名` = 「システム」（完全一致）
- `列のコメント` LIKE「販売数量」 **OR** `列の注釈` LIKE「販売数量」
- 上記は **AND**（= システム注釈の付いた行の中で販売数量に該当する列を探す）

#### (E) 「製品カテゴリ別の販売数量はありますか？」
- `列` / `列のコメント` / `列の注釈` に対して「製品カテゴリ」「販売数量」を部分一致
- 条件は（各列×各語のヒットをまとめて）**OR**
- ※この形式は「列が求められている」

#### (F) 「顧客セグメント別の実績表はありますか？（または 実績はどこですか？）」
- 列側：`列` / `列のコメント` / `列の注釈` LIKE「顧客セグメント」（部分一致）
- 表側：`表` / `表の注釈` LIKE「実績」（部分一致）
- 列側グループと表側グループは **AND**
- ※この形式は「列が求められている」

#### (G) フォールバック例：「販売金額を探して」「販売金額はどこですか」「販売金額はありますか」
- `列` LIKE「販売金額」 **OR** `列の注釈` LIKE「販売金額」
- ※この形式は「列が求められている」

---

## 4. ユーザーが出力列を指示した場合の扱い
- 指示に従って出力列を選ぶ。
- ただし **`所有者` は必ず表示**する。
- 例：
  - 「所有者がZZZの表だけ表示して」→ `所有者` / `表`
  - 「XXX表に含まれる列だけを表示して」→ `所有者` / `表` / `列`
  - 「XXX表に含まれる列を表示して」→ 3-2 のデフォルト（列情報を含めて出す）
